## АЛГОРИТМ САМОИСПРАВЛЕНИЯ (Self-Healing Loop)

Если какой-то пункт выдает FAIL:

```
1. ИИ логирует ошибку: { function: "...", input: {...}, error: "...", timestamp: "..." }
2. ИИ читает CandidateLog/AgentTicket/Entity за последние 60 секунд
3. ИИ анализирует: почему произошла ошибка?
   - Неверный ID? → Переспрашиваем ID у пользователя
   - Недостаточно прав (403)? → Выводим: "У вас нет прав (role=moderator), требуется admin"
   - Координаты отсутствуют? → Предлагаем syncAssemblyPoints
   - Функция вернула NULL? → Проверяем валидность входных данных
4. Если это исправимо:
   - ИИ предлагает действие: "Я нашел проблему: [описание]. Совет: [решение]"
   - Получаем согласие пользователя
   - Повторяем функцию с исправленными параметрами
5. Если это не исправимо → документируем как "ИЗВЕСТНОЕ ОГРАНИЧЕНИЕ"
6. Перезапускаем тест для этого пункта и выводим результат (PASS / FAIL с причиной)
```

**Пример цикла исправления:**
```
FAIL: getCandidateLogistics для "Test Candidate 1"
Лог: { error: "Координаты города кандидата не найдены в справочнике" }

Анализ ИИ:
- Город "Хабаровск" есть в Candidate
- Но в City не найдена запись с name="Хабаровск" и lat/lon
- Решение: вызвать syncAssemblyPoints

Предложение пользователю:
"Я обнаружил, что город 'Хабаровск' отсутствует в справочнике городов. 
Решение: вызвать syncAssemblyPoints для обновления справочника. Выполнить?"

Пользователь: "Да"

Исправление:
- Вызываем syncAssemblyPoints
- Проверяем, что "Хабаровск" добавлен в City с координатами
- Повторяем getCandidateLogistics

Результат: PASS ✓
```

## ПРОВЕРКА ПРАВ (Permissions Audit)

### Сценарий 1: Несанкционированная операция (тест безопасности)
```
Попытка: ИИ пытается вызвать City.delete с moderator
Ожидаемый результат: 
  - Функция выполняется, так как у агента есть permission на City.delete в tool_configs
  - Но внутри функции должна быть проверка: if (user.role !== 'admin') return 403
  
Если функция позволит модератору удалить город → FAIL (уязвимость)
Если функция вернёт 403 → PASS (защита работает)
```

```
  - Moderator получает: { error: "Only admins can respond to tickets", status: 403 }
  
PASS: Функция блокирует доступ ✓
```
